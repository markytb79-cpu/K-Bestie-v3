목적:
ChatGPT가 현재 구현된 내친구 케이 화면을 직접 열어 확인할 수 있도록 임시 검토 모드를 만들어주세요.

현재 서비스는 아직 오픈 전이며 모든 데이터는 테스트 데이터입니다.
따라서 기존 화면, 기존 테스트 데이터, 기존 컴포넌트를 그대로 사용해도 됩니다.

핵심 요구사항:

1. 로그인 없이 임시 검토 경로로 접속 가능하게 해주세요.

예:
https://app.k-bestie.com/chatgpt-review-임시토큰

2. 해당 경로에 접속하면 현재 서비스의 실제 화면을 확인할 수 있게 해주세요.

- 부모 화면
- 아이 화면
- 운영자 화면
- 현재 테스트 데이터 그대로 사용
- 기존 UI와 실제 컴포넌트 그대로 사용
- 새 디자인이나 별도 목업을 만들지 않음

3. 검토용 첫 화면에 전체 화면 목록을 만들어주세요.

공통:
- 통합 로그인 화면
- 부모 로그인 화면
- 아이 로그인 화면

부모:
- 부모 홈
- 자녀 프로필 관리
- 자녀 등록
- 자녀 수정
- 자녀 삭제
- 복구 요청
- 리포트 목록
- 리포트 상세
- 오늘의 대화거리
- 부모 질문 등록
- 케이와의 놀이 이용 기록
- 설정

아이:
- 아이 홈
- 미션 화면
- 자유 대화
- 황금열쇠 화면
- 케이와의 놀이 선택
- 놀이 진입 팝업
- 이어하기 및 새로 시작
- 놀이 세션 만료 화면

운영자:
- 관리자 대시보드
- 사용자 관리
- 자녀 데이터 복구 요청
- 안전 이벤트
- 인공지능 설정
- 감사 로그

각 항목을 누르면 해당 실제 화면으로 직접 이동할 수 있게 해주세요.

4. 인증 우회 방식

- 기존 운영 인증 로직을 삭제하거나 변경하지 않음
- 검토용 경로에서만 임시 테스트 부모·아이·운영자 세션을 주입
- 부모, 아이, 운영자 역할을 화면에서 전환 가능하게 구성
- 일반 서비스 주소에서는 기존 로그인 방식 유지

5. ChatGPT가 읽을 수 있도록 처리

- 각 화면 URL에 직접 접속 가능
- 로그인 확인 중, 사용자 정보 확인 중, 불러오는 중 화면에서 멈추지 않음
- 핵심 화면 텍스트와 UI가 초기 접근 시 바로 렌더링
- 클라이언트 인증 완료 후에만 화면이 나타나는 방식은 검토 경로에서 우회
- 각 화면의 직접 URL 목록 제공

6. 테스트 데이터 사용

현재 DB의 테스트 데이터 사용 가능.
실제 서비스에 존재하는 테스트 부모, 테스트 자녀, 테스트 리포트, 황금열쇠, 놀이 이용 기록을 그대로 표시해주세요.

단, 검토 과정에서 실수로 데이터가 변경되지 않도록 다음 쓰기 기능은 비활성화해주세요.

- 실제 삭제
- 실제 탈퇴
- 동의 철회
- 결제
- 복구 승인
- 데이터 영구 변경

화면 이동, 팝업 표시, 입력 화면 확인은 가능하게 해주세요.

7. 임시 기능 관리

- 환경변수 CHATGPT_REVIEW_MODE=true일 때만 활성화
- 추측하기 어려운 임시 경로 또는 토큰 사용
- noindex, nofollow 적용
- 검토 종료 후 환경변수만 끄면 접근 불가
- 운영 인증과 일반 사용자 화면에는 영향 없음
