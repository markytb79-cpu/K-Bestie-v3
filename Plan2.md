관리자 대시보드 "사용량·비용" 탭을 실제 비용 모니터링 도구 수준으로 다시 만들어줘. 지금은 건수만 나열된 뼈대라 못 쓴다. 목적은 STT / TTS / Gemini Live 세 가지의 사용량·비용을 한 화면에서 비교·모니터링하는 것. 개발자용 세부지표(P99 등)는 넣지 마.

[데이터 소스 — 서비스마다 다름]
1) STT/TTS (Google Cloud 서비스): 실제 청구액을 GCP billing_export(BigQuery, 프로젝트 K-Bestie3, 데이터세트 billing_export)에서 SKU별로 가져온다. BigQuery 연동해줘. 조회용 서비스 계정 인증에 필요한 권한/키를 먼저 한 줄로 정리해서 알려주고, 내가 GCP에서 직접 만들어야 할 게 있으면 짚어줘.
2) Gemini Live API (AI Studio): 실제 청구액을 프로그램으로 가져올 공식 export가 없다(확인 완료). 대신 usage/live 라우트에서 재는 세션 시간(초)과 API 응답 usageMetadata(prompt_token_count, response/candidates token 등)를 usage_events에 저장하고 공식 단가로 계산한다. Live 오디오는 25 tokens/초 기준, 입력 오디오 $3.00/1M 토큰(≈$0.005/분), 출력 오디오 $12.00/1M 토큰(≈$0.018/분). 이 단가는 pricing.ts 상수로 넣어 나중에 바꾸기 쉽게.

[화면 구성]
1) 상단 기간 필터: 오늘 / 최근 7일 / 최근 30일. 선택 기간 기준으로 아래 전부 갱신.
2) 비교 요약 카드(상단): "우리 추정 비용 합계" vs "GCP 실제 청구액 합계(STT/TTS)"를 큰 숫자로 나란히 + 차이(원, %). 제미나이 라이브는 실제값 자동 연동이 안 되므로 "계산 비용"으로 별도 카드에 표기하고 "AI Studio Billing에서 월말 대조 필요" 안내 문구.

4) 일별 추이 그래프: 날짜(x축)×비용(y축). STT/TTS는 추정선과 실제 청구선을 겹쳐서(벌어지면 추정 오차가 보이게), Live는 계산 비용선. 차트 라이브러리는 프로젝트에 이미 있는 것 우선, 없으면 recharts 가볍게 도입.
5) 트래픽 현황: 기간 내 총 대화 세션 수, STT/TTS/Live 각 호출 건수.

[공통]
