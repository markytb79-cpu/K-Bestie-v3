### 📋 작업 지시서 — [대상: Claude Code / 랄프(자율) 모드]

현재 대부분 화면이 정적 폼·더미 표시 상태다. 이제 각 화면의 기능이 실제로 동작하도록 구현한다. 1번부터 순서대로 끝까지 진행하고, STOP 조건 외에는 멈추지 마라. 완료 시 기능별 동작 상태표와 변경 파일 목록을 보고하라.

#### 범위 원칙
- 음성 대화·자막(Live API) 로직은 이번 범위에서 제외(건드리지 말 것).
- 가입·로그인·결제는 더미 유지. 그 외 폼·버튼·상태·데이터 연동은 실제로 동작하게.
- 데이터 저장은 Supabase 사용. 단 신규 테이블/컬럼이 필요하면 **마이그레이션 SQL 파일만 생성**하고 실제 실행은 하지 마라(STOP). 마이그레이션 적용 전까지는 localStorage 기반 데모 스토어로 동작시켜, 적용 후 Supabase로 스위치할 수 있게 데이터 레이어를 분리(lib/store)하라.

#### 1. 아이 추가 폼 동작
- 이름 입력(최대 10자), 학년 선택(단일), 관심사 다중 선택이 상태로 묶이고, "아이 추가 완료" 시 데모 스토어에 아이 레코드 저장 → /parent/home 이동.
- 저장된 아이가 부모 홈 프로필 스위처와 설정 "자녀 관리"에 즉시 반영. 검증: 이름 미입력·학년 미선택 시 버튼 비활성 또는 안내.

#### 2. 부모 홈 프로필 스위처
- 등록된 아이 목록을 스위처에 렌더, 선택 시 active child 전환 → 홈의 "OO이의 오늘" 카드·지표가 선택된 아이 기준으로 바뀐다. "+추가"는 아이 추가로 연결(기존).

#### 3. 부모 질문 등록/관리 (핵심 기능)
- 부모가 질문 입력 → 큐에 "대기 중"으로 추가. 질문별 상태(대기 중/전달됨/중지됨) 토글·중지 동작. 등록 시각·전달 횟수 표시.
- /parent/guide와 /parent/settings의 "부모 질문 관리"가 같은 데이터 소스를 공유(한쪽에서 등록하면 양쪽 반영).

#### 4. 아이 미션 동작
- /child/missions, /child/home의 미션 체크가 상태로 동작: 미션 완료 토글 시 "n/5 완료" 카운트·진행바 갱신, 완료 목록/남은 목록 재분류. /child/finish의 완료 개수도 같은 소스 반영.

#### 5. 기분 별점 / 감정 입력
- "오늘 기분 별점 남기기" 미션에서 별점 선택 → 저장되고 미션 완료 처리. (부모 리포트의 기분 점수와 데이터 소스 연결 구조만 잡아두고, 값은 데모 기준.)

#### 6. 알림 / 설정 토글
- /parent/settings의 알림 토글(리포트·감정위험·주간요약)이 실제 on/off 상태로 저장·반영.
- /parent/notifications 알림 항목 읽음 처리 → 헤더·탭 뱃지 카운트 감소. 뱃지 숫자와 실제 미읽음 개수 일치.

#### 7. 로그아웃(데모 초기화) 동작
- 데모 스토어(아이·질문·미션·설정) 전부 비우고 / 로 이동. 초기화 후 부모 홈은 "등록된 아이 없음" 상태로 복귀.

#### 8. 데이터 레이어 정리
- 모든 데모 데이터를 lib/store(또는 lib/demo-data + 상태 훅) 단일 소스로 통합. 화면들은 이 소스만 읽고 쓴다. Supabase 스위치 지점을 주석으로 명시.

#### 공통

- npm run build / tsc --noEmit 통과 유지.
