# 내친구 케이 질문·대화 엔진 구현 명세

## Claude Code 전달용 — v1.0

- 대상 프로젝트: 내친구 케이(K-Bestie)
- 기술 전제: Next.js + TypeScript + Supabase 기반의 현재 코드베이스
- 기준 문서: 「내친구 케이 아동 대화 질문지 — 아동 심리 상담사 검토용 v2.0」
- 구현 목표: 검토·승인된 질문만 사용해 아이와 자연스럽게 대화하고, 답변을 부모 앱의 8개 리포트 항목에 근거 데이터로 연결한다.
- 최우선 원칙: AI가 질문 목적을 임의로 만들거나 승인되지 않은 질문으로 아이를 탐색하지 않는다.

---

# 1. 결론 및 구현 방향

## 채택안: 승인된 질문은행 기반의 결정론적 질문 엔진

초기 알파·베타에서는 LLM이 질문을 새로 생성하지 않는다.

1. 서버가 질문은행에서 사용할 질문을 결정한다.
2. LLM은 선택된 질문을 아이의 직전 대화에 자연스럽게 연결한다.
3. LLM은 질문의 뜻, 범위, 의도를 바꾸지 않는다.
4. 아이 답변에 대한 꼬리 질문도 승인된 질문 또는 승인된 꼬리 질문 규칙에서만 선택한다.
5. 부모가 등록한 관심사는 아이에게 새 질문을 만드는 데 사용하지 않고, 리포트에서 기존 답변과 매칭하는 데만 사용한다.

## 보류안: 의도 기반 동적 질문 생성

LLM이 `질문 주제 + 질문 의도`를 받아 새로운 질문을 생성하는 방식은 상담사 검토와 실제 사용 데이터가 축적된 후 별도 단계에서 검토한다. 현재 버전에는 적용하지 않는다.

# 2. 구현 범위

이번 작업 범위는 다음과 같다.

1. 질문은행 데이터 구조
2. 질문지 v2.0의 구조화 및 초기 데이터 입력
3. 질문 선택·순서·중복 방지 로직
4. 관심사 → 일상 → 감정 대화 흐름
5. 매일·매주·분기·첫 만남 질문 운영
6. 아이 답변 유효성·회피·무응답 처리
7. 승인된 꼬리 질문 1회 처리
8. 부모 앱 8개 리포트 항목과 답변 근거 연결
9. 형제자매 간 데이터 분리
10. 상담·법률 검토 전 기능의 비활성화 장치
11. 테스트와 운영 검증 로그

이번 작업에서 하지 않는 것:

- 심리 진단
- 성격·정서 상태의 자동 판정
- 부모 요청을 아이 질문으로 변환
- 질문 목적의 자유 생성
- 상담사 미승인 위기 대응 문구의 운영 적용
- 아이 답변을 근거로 친구·교사·부모를 평가
- 아이의 답변 원문을 다른 자녀 프로필이나 리포트에 공유

# 3. 부모 앱 8개 리포트 주제 코드

질문과 답변은 반드시 아래 주제 중 하나 이상에 연결한다.

| 코드 | 부모 앱 표시명 | 주요 근거 데이터 |
|---|---|---|
| `SCHOOL_ACADEMY` | 학교·학원 생활 | 수업, 쉬는 시간, 점심, 학원, 교사, 장소, 사건, 아이 반응 |
| `PEER_RELATIONSHIP` | 친구 관계와 또래 생활 | 친구, 함께한 활동, 관계 변화, 갈등·협력 상황, 아이 감정 |
| `EMOTION_HINT` | 감정 힌트 | 아이가 직접 사용한 감정 표현, 감정 발생 상황, 감정 변화 |
| `PERSONAL_INTEREST` | 관심사와 개인 취향 | 놀이, 음식, 장소, 캐릭터, 활동, 좋아하는 이유, 취향 변화 |
| `STUDY_CONCERN` | 공부 고민 | 과목, 과제, 어려운 지점, 자신 있는 부분, 원하는 도움 |
| `DIGITAL_CONTENT` | 디지털 관심사와 콘텐츠 취향 | 영상, 게임, 만화, 캐릭터, 이용 상황, 좋아하는 이유 |
| `FUTURE_DREAM` | 미래·진로·꿈 | 배우고 싶은 것, 해보고 싶은 일, 기대, 되고 싶은 모습 |
| `REPEATED_STORY` | 반복되는 이야기 | 일정 기간 반복 등장한 사람, 사건, 활동, 감정, 어려움 |

`REPEATED_STORY`는 아이에게 직접 묻는 기본 주제가 아니라, 다른 주제의 누적 답변에서 반복 패턴을 탐지해 생성한다. 확인이 필요한 경우에만 승인된 기억 연결 질문을 사용한다.

# 4. 질문 데이터 모델

기존 코드베이스와 Supabase 구조를 먼저 확인한 후, 중복 테이블이 없다면 아래 구조를 기준으로 구현한다.

## 4.1 `question_groups`

질문의 고정된 목적을 관리한다. 문장 변형이 있어도 같은 목적이면 같은 그룹에 속한다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `id` | uuid | 질문 그룹 ID |
| `code` | text unique | 예: `DAILY_MAIN_EVENT`, `PEER_RECENT_FRIEND` |
| `title` | text | 관리자용 그룹명 |
| `main_topic` | enum/text | 8개 리포트 주제 코드 |
| `sub_topic` | text | 세부 주제 |
| `conversation_stage` | enum | `INTEREST`, `DAILY_LIFE`, `EMOTION`, `BRIDGE`, `FOLLOW_UP` |
| `question_intent` | text | 어떤 답변을 확보하려는지 구체적으로 기술 |
| `expected_answer_elements` | jsonb | 사람, 장소, 사건, 감정, 이유 등 |
| `minimum_valid_answer` | text | 유효 답변 최소 조건 |
| `report_usage` | text | 부모 리포트 활용 방식 |
| `frequency` | enum | `FIRST_MEETING`, `DAILY_FIXED`, `DAILY_ROTATION`, `WEEKLY_FIXED`, `MONTHLY`, `QUARTERLY`, `CONTEXTUAL` |
| `preferred_time` | enum nullable | `AFTER_SCHOOL`, `BEDTIME`, `WEEKEND`, `ANY` |
| `age_min` | int | 기본 9 |
| `age_max` | int | 기본 10 |
| `clinical_status` | enum | `DRAFT`, `PENDING_REVIEW`, `APPROVED`, `REJECTED` |
| `is_active` | boolean | 운영 사용 여부 |
| `source_version` | text | `QUESTIONNAIRE_V2_0` |
| `created_at` | timestamptz | 생성일 |
| `updated_at` | timestamptz | 수정일 |

## 4.2 `question_variants`

아이에게 실제로 들려줄 문장과 대체 표현을 관리한다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `id` | uuid | 질문 문장 ID |
| `question_group_id` | uuid | 질문 그룹 |
| `question_text` | text | 실제 질문 |
| `variant_type` | enum | `CANONICAL`, `ALTERNATIVE`, `FOLLOW_UP`, `POSITIVE_CLOSE` |
| `tone` | enum | `NEUTRAL`, `PLAYFUL`, `CALM` |
| `requires_context` | boolean | 직전 답변이나 과거 기록 필요 여부 |
| `context_requirements` | jsonb | 필요한 인물명, 활동명, 이전 사건 등 |
| `repeat_cooldown_days` | int | 동일 문장 재사용 제한 |
| `clinical_status` | enum | 문항별 검토 상태 |
| `is_active` | boolean | 실제 사용 여부 |
| `sort_order` | int | 기본 순서 |

## 4.3 `question_transitions`

관심사 → 일상 → 감정 흐름을 정의한다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `from_group_id` | uuid | 현재 질문 그룹 |
| `to_group_id` | uuid | 다음 질문 그룹 |
| `transition_type` | enum | `DEFAULT`, `VALID_ANSWER`, `REFUSAL`, `NO_RESPONSE`, `CONTEXT_KNOWN` |
| `condition` | jsonb | 전환 조건 |
| `priority` | int | 우선순위 |

## 4.4 `child_question_state`

아이별 질문 이력과 현재 흐름을 관리한다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `child_id` | uuid | 자녀 ID |
| `question_group_id` | uuid | 질문 그룹 |
| `last_asked_at` | timestamptz | 마지막 질문 시각 |
| `last_variant_id` | uuid | 마지막 문장 |
| `ask_count_7d` | int | 최근 7일 질문 횟수 |
| `ask_count_30d` | int | 최근 30일 질문 횟수 |
| `last_answer_status` | enum | `VALID`, `PARTIAL`, `REFUSAL`, `NO_RESPONSE`, `SAFETY_SIGNAL` |
| `last_answer_summary` | text nullable | 답변의 최소 요약 |
| `updated_at` | timestamptz | 갱신 시각 |

이 테이블은 반드시 `child_id` 기준으로 분리한다. 서아의 질문 이력·답변·리포트 근거가 서현에게 연결되거나 그 반대가 발생하면 안 된다.

## 4.5 `conversation_question_events`

실제 대화에서 어떤 질문이 사용됐는지 감사 로그로 남긴다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `id` | uuid | 이벤트 ID |
| `session_id` | uuid | 대화 세션 |
| `child_id` | uuid | 자녀 |
| `question_group_id` | uuid | 질문 목적 |
| `question_variant_id` | uuid | 실제 문장 |
| `sequence_no` | int | 질문 순서 |
| `selection_reason` | text | 고정, 부족 주제 보완, 주간, 문맥 연결 등 |
| `asked_text` | text | 최종 발화 문장 |
| `answer_status` | enum | 답변 상태 |
| `created_at` | timestamptz | 시각 |

## 4.6 `answer_evidence`

답변에서 부모 리포트용 근거만 구조화한다.

| 필드 | 형식 | 설명 |
|---|---|---|
| `id` | uuid | 근거 ID |
| `session_id` | uuid | 세션 |
| `child_id` | uuid | 자녀 |
| `question_event_id` | uuid | 질문 이벤트 |
| `main_topic` | text | 8개 리포트 주제 |
| `sub_topic` | text | 세부 주제 |
| `event_summary` | text nullable | 사건 요약 |
| `people` | jsonb | 등장인물 |
| `places` | jsonb | 장소 |
| `activities` | jsonb | 행동·활동 |
| `child_emotion_words` | jsonb | 아이가 직접 사용한 감정 표현 |
| `preference_reason` | text nullable | 좋아하거나 불편한 이유 |
| `desired_support` | text nullable | 원하는 도움 |
| `future_intent` | text nullable | 앞으로 하고 싶은 일 |
| `confidence` | numeric | 추출 신뢰도 |
| `is_report_eligible` | boolean | 리포트 근거 사용 가능 여부 |
| `created_at` | timestamptz | 생성일 |

금지:

- `diagnosis`
- `risk_score`를 일반 감정 분석에 사용
- `personality_type`
- `attachment_style`
- `addiction_probability`
- 친구·부모·교사의 평가 점수

# 5. 질문 승인 상태와 기능 플래그

질문은 상담사 판정 결과에 따라 다음 상태를 가진다.

- `DRAFT`: 내부 초안
- `PENDING_REVIEW`: 상담사 검토 대기
- `APPROVED`: 실제 아이 대화에 사용 가능
- `REJECTED`: 사용 금지

운영 질문 선택 쿼리는 반드시 다음 조건을 만족해야 한다.

```text
clinical_status = 'APPROVED'
AND is_active = true
```

초기 시드 입력 시 질문지 v2.0의 일반 문항은 모두 `PENDING_REVIEW`로 입력한다. 상담사가 승인한 문항만 관리 화면 또는 승인 스크립트를 통해 `APPROVED`로 전환한다.

위기 신호 관련 질문·반응·처리는 별도의 기능 플래그를 둔다.

```text
QUESTION_ENGINE_V2=true
CLINICAL_QUESTION_BANK_ENABLED=false
CRISIS_FLOW_V2_ENABLED=false
PARENT_INTEREST_TO_QUESTION_ENABLED=false
```

- 상담사 승인 전 `CLINICAL_QUESTION_BANK_ENABLED=false`
- 위기 대응 문구와 통지 경로의 임상·법률 검토 전 `CRISIS_FLOW_V2_ENABLED=false`
- 부모 관심사를 아이 질문으로 변환하는 기능은 설계상 계속 `false`

`CRISIS_FLOW_V2_ENABLED=false` 상태에서도 기존 서비스의 기본 안전 정책을 제거하거나 약화하지 않는다. 현재 운영 중인 안전 처리 로직을 유지하고, 새 초안 규칙만 비활성화한다.

# 6. 질문 선택 엔진

## 6.1 공통 선택 우선순위

1. 안전 정책
2. 현재 세션의 필수 고정 질문
3. 직전 답변과 자연스럽게 연결되는 승인된 다음 단계 질문
4. 최근 데이터가 부족한 부모 리포트 주제
5. 주간·월간·분기 질문
6. 첫 만남 친해지기 질문
7. 최근 사용하지 않은 로테이션 질문
8. 긍정 마무리 질문

## 6.2 질문 선택 금지 조건

다음 질문은 선택하지 않는다.

- `clinical_status != APPROVED`
- 동일 문장 쿨다운 기간 미경과
- 아이가 직전 대화에서 같은 의도를 충분히 답한 질문
- 아이가 해당 주제에 대해 말하기 싫다고 한 직후의 유사 질문
- 취침 전 불안을 확장할 가능성이 있어 승인 시점 제한이 설정된 질문
- 필요한 이전 문맥이 없는데 과거 이야기를 전제로 하는 질문
- 다른 자녀의 기록을 문맥으로 사용하는 질문
- 부모 관심사를 근거로 새로 생성된 질문

## 6.3 부모 리포트 데이터 부족도

질문을 카드 채우기 목적으로 기계적으로 반복하지 않는다. 다만 일정 기간 데이터가 없는 주제는 로테이션 후보 우선순위를 높일 수 있다.

권장 기본 기간:

| 주제 | 데이터 확인 기간 |
|---|---|
| 학교·학원 | 최근 7일 |
| 친구 관계 | 최근 7일 |
| 감정 힌트 | 최근 7일 |
| 관심사 | 최근 30일 |
| 공부 고민 | 최근 30일 |
| 디지털 콘텐츠 | 최근 30일 |
| 미래·꿈 | 최근 90일 |
| 반복 이야기 | 최근 30일 누적 분석 |

데이터가 없다는 이유로 동일 주제를 같은 세션에서 반복 추궁하지 않는다.

# 7. 세션 구성

현재 서비스의 하루 2회 미션 구조에 연결한다.

## 7.1 하교 후 미션

최대 5개 질문을 기본으로 한다.

1. 오늘의 대표 사건 고정 질문
2. 현재 감정 고정 질문
3. 학교·학원 또는 친구 관계 로테이션
4. 관심사·공부·디지털 중 데이터가 부족한 주제
5. 승인된 꼬리 질문 1개 또는 긍정 마무리

## 7.2 취침 전 미션

최대 5개 질문을 기본으로 한다.

1. 하루 돌아보기 고정 질문
2. 긍정 경험 또는 편안했던 순간
3. 오늘 대화에서 충분히 다루지 않은 주제
4. 내일의 기대 질문
5. 긍정 마무리

`걱정되는 일`을 취침 전에 묻는 문항은 상담사 승인과 시간대 조건이 확정되기 전까지 사용하지 않는다.

## 7.3 첫 만남 질문

첫 2주 동안 하루 1~2개만 섞는다. 한 세션 전체를 프로필 조사처럼 구성하지 않는다.

우선순위:

1. 원하는 호칭
2. 좋아하는 대화 방식
3. 거절 표현
4. 최근 관심사
5. 케이가 기억해주길 원하는 내용

비밀보장 한계 관련 문구는 임상·법률 검토 전 사용하지 않는다.

## 7.4 주간 질문

토요일과 일요일에 나누어 최대 2개씩 사용한다. 질문지에 적힌 8개를 한 번에 모두 묻지 않는다.

## 7.5 분기 질문

3개월마다 2~3회 세션으로 나누고, 세션당 최대 2개만 사용한다. 기존 대화에서 이미 충분히 확인된 문항은 생략한다.

---

# 8. 관심사 → 일상 → 감정 흐름

질문은 같은 주제를 다음 순서로 확장한다.

```text
INTEREST → DAILY_LIFE → EMOTION
```

예시:

```text
요즘 가장 자주 하는 게임이 뭐야?
→ 그 게임은 보통 언제 하고 누구와 같이 해?
→ 그 게임에서 어떤 부분이 제일 재미있어?
```

## 흐름 생략 조건

직전 답변이나 최근 기록에서 필요한 정보가 이미 확인된 경우 해당 단계를 생략한다.

예:

- 아이가 첫 답변에서 게임 이름, 친구와 함께한 상황, 재미있는 이유까지 모두 말한 경우 추가 질문 없이 다음 주제로 이동
- 오늘 이미 학교 사건과 감정이 충분히 확인된 경우 같은 내용을 다시 묻지 않음
- 과거 기록만 있고 오늘 상황이 불명확하면 `BRIDGE` 질문 사용

## 단계별 목적

| 단계 | 목적 |
|---|---|
| `INTEREST` | 아이가 좋아하거나 관심을 갖는 대상을 쉽게 말하게 함 |
| `DAILY_LIFE` | 언제, 어디서, 누구와, 무엇을 했는지 실제 경험을 확인 |
| `EMOTION` | 아이가 직접 느낀 감정, 좋아한 이유, 불편한 이유, 바람을 확인 |
| `BRIDGE` | 과거 관심사와 오늘의 일상을 자연스럽게 연결 |
| `FOLLOW_UP` | 직전 답변의 핵심 한 가지를 한 단계만 구체화 |

# 9. LLM 역할 분리

하나의 LLM 호출이 질문 선택, 대화, 안전 판정, 리포트 생성을 모두 수행하지 않도록 분리한다.

## 9.1 서버 질문 오케스트레이터

역할:

- 세션 유형 결정
- 질문 그룹 선택
- 승인 상태·쿨다운·질문 이력 확인
- 다음 단계 전환
- 질문 수 제한
- 형제자매 데이터 분리

LLM이 아닌 서버 로직이 담당한다.

## 9.2 케이 대화 응답기

입력:

- 선택된 질문 문장
- 질문 목적
- 직전 아이 발화
- 필요한 최소 문맥
- 허용된 꼬리 질문
- 말투 설정

역할:

- 아이 발화를 1~2문장으로 자연스럽게 받아줌
- 서버가 선택한 승인 질문을 이어서 말함
- 질문의 목적과 범위를 바꾸지 않음
- 새로운 탐색 질문을 임의로 추가하지 않음

## 9.3 답변 분류기

출력 상태:

- `VALID`
- `PARTIAL`
- `REFUSAL`
- `NO_RESPONSE`
- `SAFETY_SIGNAL`

답변 분류와 안전 신호 분류는 분리된 필드로 반환하는 편이 좋다.

## 9.4 리포트 근거 추출기

아이 답변에서 확인된 사실만 추출한다.

추출 가능:

- 사람
- 장소
- 사건
- 행동
- 과목
- 콘텐츠
- 아이가 직접 말한 감정
- 좋아하는 이유
- 어려운 지점
- 원하는 도움
- 앞으로 하고 싶은 일

추출 금지:

- 진단
- 성격 단정
- 숨은 의도 추정
- 친구 관계의 좋고 나쁨 판정
- 부모 양육 평가
- 과몰입·중독 판정

# 10. 케이 대화 응답기 시스템 프롬프트

아래 프롬프트를 실제 코드 구조에 맞게 상수 또는 프롬프트 템플릿으로 관리한다.

```text
너는 초등학교 4학년 아이와 대화하는 AI 친구 ‘케이’다.

너의 역할은 아이의 말을 편안하게 듣고, 서버가 전달한 승인된 질문을 자연스럽게 이어서 묻는 것이다.

반드시 지킬 규칙:

1. 서버가 전달한 selected_question의 의미, 주제, 의도를 바꾸지 않는다.
2. selected_question에 없는 새로운 탐색 질문을 만들지 않는다.
3. 아이의 직전 답변을 짧게 받아준 뒤 selected_question을 한 번만 묻는다.
4. 한 번에 질문은 하나만 한다.
5. 아이의 말을 평가, 훈계, 비교, 진단하지 않는다.
6. 친구, 부모, 교사의 잘잘못을 판단하지 않는다.
7. ‘왜?’만 단독으로 묻지 않는다.
8. 아이가 ‘몰라’, ‘그냥’, ‘비밀이야’, ‘말하기 싫어’, ‘기억 안 나’, ‘귀찮아’라고 하면 압박하지 않는다.
9. 아이가 말하기 싫다고 하면 ‘알겠어. 말하고 싶을 때 이야기해도 돼.’처럼 짧게 받아주고 서버가 전달한 전환 문장으로 이동한다.
10. 과거 대화를 사용할 때는 반드시 current_child_id의 확인된 기록만 사용한다.
11. 다른 자녀나 다른 사용자의 정보를 절대로 섞지 않는다.
12. 아이가 직접 말하지 않은 감정이나 이유를 추측하지 않는다.
13. 부모가 궁금해한다는 사실을 아이에게 말하지 않는다.
14. 질문을 통해 부모 대신 사실을 캐묻지 않는다.
15. selected_question이 없으면 새로운 질문을 만들지 말고 자연스럽게 대화를 마무리한다.

출력 형식:

{
  "acknowledgement": "아이의 직전 말을 받아주는 1~2문장",
  "question": "selected_question과 동일한 목적의 승인된 질문 문장 또는 빈 문자열",
  "combined_speech": "실제로 아이에게 들려줄 전체 문장"
}
```

LLM이 문장을 자유롭게 바꾸도록 허용하지 않을 경우 `question`은 서버에서 전달된 문장을 그대로 사용하고, LLM은 `acknowledgement`만 생성하게 한다. 초기 알파에서는 이 방식을 권장한다.

# 11. 답변 판정 규칙

## 11.1 유효 답변

다음 중 하나 이상이 구체적으로 포함되면 `VALID` 후보로 판단한다.

- 사람
- 장소
- 사건
- 활동
- 물건
- 과목
- 콘텐츠
- 감정
- 좋아하거나 불편한 이유
- 원하는 도움
- 앞으로 하고 싶은 행동

예:

- “민지랑 피구했어.”
- “수학 분수가 어려웠어.”
- “마인크래프트에서 집 만드는 게 좋아.”
- “발표할 때 조금 떨렸어.”

질문별 `minimum_valid_answer` 조건을 최종 기준으로 사용한다.

## 11.2 부분 답변

주제와 관련은 있으나 질문 의도에 필요한 핵심 요소가 부족하면 `PARTIAL`.

예:

- 질문: “누구와 무엇을 했어?”
- 답변: “친구랑.”

`PARTIAL`이라고 해서 반드시 꼬리 질문을 하지 않는다. 세션 길이, 아이 반응, 당일 질문 수를 함께 고려한다.

## 11.3 회피

다음 표현은 `REFUSAL` 또는 회피로 처리한다.

- 몰라
- 그냥
- 없어
- 비밀이야
- 말하기 싫어
- 기억 안 나
- 귀찮아

처리:

1. 짧게 수용
2. 같은 질문 재시도 금지
3. 유사 의도의 질문도 현재 세션에서 제외
4. 부담이 낮은 다른 주제 또는 긍정 마무리로 전환

## 11.4 무응답

기존 확정 정책에 따라 5초 이상 침묵하면 한 번만 쉬운 표현으로 재안내한다. 다시 무응답이면 다음 질문으로 이동한다.

질문의 핵심 목적을 바꾸는 재질문은 금지한다.

---

# 12. 꼬리 질문 규칙

1. 한 기본 질문당 최대 1회
2. 질문지 v2.0의 승인된 꼬리 질문만 사용
3. 직전 답변의 핵심 한 가지에만 연결
4. 이미 필요한 정보가 확보됐으면 생략
5. 아이가 회피·무응답·불편함을 보이면 사용 금지
6. 안전 신호 가능성이 있으면 일반 꼬리 질문 중단
7. 다음 표현은 사용 금지

```text
왜?
그래서?
정확히 말해봐.
그것밖에 없어?
누가 먼저 그랬어?
진짜야?
부모님에게 말했어?
네가 잘못한 건 없어?
```

# 13. 부모 관심사 처리

부모 관심사는 아이 질문 생성 입력으로 사용하지 않는다.

## 허용 흐름

```text
부모 관심사 등록
→ 평상시 승인 질문으로 아이와 대화
→ 아이가 자연스럽게 관련 내용을 말함
→ 답변 근거와 부모 관심사를 리포트 단계에서 매칭
→ 아이가 말한 범위 안에서 부모에게 요약
```

## 금지 흐름

```text
부모 관심사 등록
→ LLM이 중립적으로 보이는 질문 생성
→ 케이가 부모 대신 아이에게 질문
```

부모 관심사 매칭 테이블이 필요하다면 다음 정도로 제한한다.

| 필드 | 설명 |
|---|---|
| `parent_interest_id` | 부모 관심사 |
| `child_id` | 대상 자녀 |
| `topic_codes` | 관련 8개 주제 |
| `keywords` | 리포트 매칭용 키워드 |
| `expires_at` | 관심사 만료 |
| `matched_evidence_ids` | 자연 발생 답변 근거 |
| `status` | 대기, 일부 확인, 확인, 만료 |

다른 자녀의 답변 근거는 매칭 대상에서 제외한다.

# 14. 반복되는 이야기 분석

질문 이벤트가 아니라 `answer_evidence`를 기준으로 탐지한다.

초기 후보 조건:

- 최근 7일 중 3일 이상 같은 주제 등장
- 최근 30일 같은 주제 5회 이상 등장
- 같은 친구·활동·과목·콘텐츠 언급이 이전 기간보다 증가
- 같은 어려움이 서로 다른 날짜에 3회 이상 등장

반복 항목은 사실만 표시한다.

허용:

- “최근 7일 중 4일 동안 민지와 함께 놀았던 이야기가 나왔어요.”
- “이번 달 수학 숙제가 어렵다는 이야기가 세 번 나왔어요.”

금지:

- “민지에게 지나치게 의존하고 있어요.”
- “수학 불안이 의심돼요.”
- “게임 중독 가능성이 있어요.”

반복 패턴 확인 질문은 질문지 v2.0의 승인된 `BRIDGE` 질문만 사용한다.

# 15. 안전·위기 관련 구현 경계

질문지 v2.0의 섹션 0은 임상 검증 전 초안이다.

따라서 이번 구현에서는 다음만 준비한다.

1. 안전 신호 분류 결과를 저장할 수 있는 인터페이스
2. 일반 질문 흐름을 즉시 중단할 수 있는 상태 전환 인터페이스
3. 위기 대응 문구와 통지 경로를 설정 파일 또는 정책 모듈로 분리
4. `CRISIS_FLOW_V2_ENABLED` 기능 플래그
5. 부모 통지와 별도 보호 경로를 분리할 수 있는 이벤트 구조
6. 감사 로그와 관리자 검토 상태

임상·법률 승인 전에는 질문지 v2.0의 위기 문구를 실제 아이에게 출력하거나 새 부모 알림 로직을 활성화하지 않는다.

기존 서비스의 안전 정책이 있다면 삭제하지 말고 그대로 유지한다. 신규 구조가 기존 정책을 대체하지 않도록 한다.

# 16. 시드 데이터 작성 규칙

질문지 v2.0을 코드에 하드코딩하지 않는다.

권장 파일:

```text
data/questions/question-bank-v2.0.json
```

또는 현재 프로젝트의 시드 관리 방식에 맞는 경로를 사용한다.

각 질문 그룹은 다음 형태로 변환한다.

```json
{
  "group_code": "DIGITAL_RECENT_CONTENT",
  "title": "최근 관심 디지털 콘텐츠",
  "main_topic": "DIGITAL_CONTENT",
  "sub_topic": "최근 자주 보는 콘텐츠",
  "conversation_stage": "INTEREST",
  "question_intent": "아이가 최근 자주 보는 영상, 만화 또는 게임의 구체적인 이름과 유형을 확인한다.",
  "expected_answer_elements": [
    "콘텐츠 이름",
    "콘텐츠 유형",
    "캐릭터 이름"
  ],
  "minimum_valid_answer": "영상, 만화, 게임, 채널 또는 캐릭터 중 하나 이상을 구체적으로 말한 답변",
  "report_usage": "최근 관심 콘텐츠를 디지털 관심사와 콘텐츠 취향 카드의 근거로 사용한다.",
  "frequency": "DAILY_ROTATION",
  "preferred_time": "ANY",
  "age_min": 9,
  "age_max": 10,
  "clinical_status": "PENDING_REVIEW",
  "source_version": "QUESTIONNAIRE_V2_0",
  "variants": [
    {
      "question_text": "요즘 가장 자주 보는 영상이나 만화, 자주 하는 게임이 뭐야?",
      "variant_type": "CANONICAL",
      "repeat_cooldown_days": 14,
      "clinical_status": "PENDING_REVIEW"
    }
  ]
}
```

질문 의도는 추상적으로 쓰지 않는다.

금지:

- 아이의 관심사를 알아본다.
- 아이의 감정을 파악한다.
- 학교생활을 확인한다.

권장:

- 아이가 오늘 학교에서 가장 기억에 남은 사건과 관련 인물을 확인한다.

# 19. 테스트 요구사항

## 19.1 단위 테스트

- 승인되지 않은 문항은 선택되지 않는다.
- 같은 문장은 쿨다운 전에 반복되지 않는다.
- 관심사 다음에 일상, 일상 다음에 감정 단계가 선택된다.
- 첫 답변에서 모든 요소가 확보되면 다음 단계를 생략한다.
- 회피 답변 후 같은 질문이나 유사 질문이 다시 선택되지 않는다.
- 꼬리 질문은 최대 1회만 사용된다.
- 취침 전 제한 문항이 상담사 승인 전 선택되지 않는다.
- 부모 관심사가 질문 생성 입력으로 전달되지 않는다.
- 다른 자녀의 기록이 문맥으로 사용되지 않는다.
- `REPEATED_STORY`는 충분한 반복 근거가 없으면 생성되지 않는다.
- 진단·평가 필드가 리포트 근거에 생성되지 않는다.

# 0. 위기 신호 감지 시 전환 규칙 (신설 — 최우선 검토)

이 섹션은 평상시 질문(섹션 1~14)에 우선한다. 아래 신호가 감지되면 케이는 진행 중이던 어떤 질문 흐름도 즉시 중단하고 이 규칙으로 전환한다.

**전환 트리거 (위기 4범주)**

1. 자기 자신을 해치는 것에 대한 언급 (우회 표현 포함: "없어지고 싶어", "사라지면 좋겠어", "잠들고 안 깨면 좋겠어", 아끼던 물건 정리, 작별 뉘앙스)
2. 다른 사람을 해치는 것에 대한 언급
3. 누군가에게 피해를 당하고 있다는 암시 (신체·정서적 학대, 성적 피해, 심한 괴롭힘)
4. 지속되는 절망·무가치감

**전환 시 절대 금지 (평상시 질문 규칙이 여기선 독이 됨)**

* 꼬리 질문·탐색 질문 금지 ("그때 무슨 일이 있었어?", "누가 같이 있었어?", "다음엔 어떻게 되면 좋겠어?" 등 섹션 14 전부 중단)
* 사건의 경위·인물·장소를 캐묻지 않는다
* 케이가 원인을 해석하거나 조언·판단하지 않는다
* "왜 그런 생각을 해?" 류 이유 추궁 금지

**전환 시 케이의 반응 원칙 (문구는 임상 검증 대상 초안)**

* 짧게 감정을 받아준다: "그 얘기 해줘서 고마워. 많이 힘들었겠다."
* 아이를 혼자 두지 않되, 케이가 해결자가 되지 않는다
* 믿을 수 있는 어른과의 연결로 이끈다: "이건 너 혼자 안고 있기엔 무거운 일이야. 믿을 수 있는 어른이랑 같이 나누면 좋겠어. 내가 도와줄게."
* 관계 초기 약속을 상기시킨다: "예전에 우리, 위험한 일은 지켜주려고 어른한테 알린다고 약속했었지."

**시스템 처리 (문항과 별개, 기술·임상 공동 설계)**

* 위기 플래그 발생 → 부모 알림 트리거 (모델 판단과 무관하게 자동)
* 가해자가 부모일 가능성이 있는 학대 정황은 부모 통지와 분리된 경로 필요 (변호사·임상 공동 설계)

**상담사 요청 사항:** 위 전환 반응 문구가 만 9~10세에게 적절한지, 안정화 효과가 있는지, 오히려 불안을 키우지 않는지 판정하고 권장 문구를 제시해 주십시오.

# 13. 부모 관심사 반영 방식 (구조 반전 — 재설계)

**변경 핵심:** 부모의 궁금증을 케이의 질문으로 심지 않는다. 부모 관심사는 아이에게 새 질문을 만들지 않고, 평상시 대화(섹션 2~12)에서 자연히 나온 내용을 **리포트 단계에서 부모 관심사와 매칭**하는 용도로만 쓴다. 케이가 부모 대리 질문자가 되면 아이 신뢰가 깨지고 조기 신호가 사라지기 때문이다.

**허용:** 부모가 "수학을 어려워하는지" 궁금하면 → 케이는 별도 질문을 하지 않고, 아이가 공부 대화(섹션 7)에서 자연히 말한 내용을 리포트에서 부모 관심사와 연결해 전달한다.

**금지:** 부모 질문을 중립 문장으로 바꿔 아이에게 직접 심는 방식(원본 13번 표 방식)은 사용하지 않는다.

**부모 요청 중 아예 수용하지 않을 항목:** 아이가 숨기는 비밀을 알아내기 위한 요청 / 거짓말 여부 확인 요청 / 특정 친구·교사를 평가하게 하는 요청 / 부모가 원하는 답을 유도하는 요청 / 처벌·통제를 위한 사실 확인 / 심리 상태·문제 행동 진단 요청.

**상담사 검토 요청:** 부모 관심사를 "질문 반영"이 아니라 "리포트 매칭"으로 처리하는 이 구조가 아동 신뢰 보호에 타당한지, 쌍둥이(서아·서현) 형제 갈등 주제를 리포트에서 다룰 때 한 아이의 사적 내용이 다른 아이 리포트로 넘어가지 않게 하는 경계 원칙을 제시해 주십시오.

# 14. 자연스러운 꼬리 질문 (평상시 전용 — 위기 시 사용 금지)

**중요:** 아래 꼬리 질문은 평상시 대화에만 쓴다. 섹션 0 위기 신호가 감지되면 이 표 전체를 중단한다.

| 상황 | 꼬리 질문 | 질문 의도 | 판정 |
|--|--|--|--|
| 사람 이름이 나온 경우 | 그 친구와는 무엇을 하면서 가장 재미있었어? | 관계·활동을 확인한다. | |
| 학교 사건이 나온 경우 | 그때 누가 같이 있었어? | 관련 인물을 확인한다. | |
| 활동이 나온 경우 | 그 활동에서 어떤 부분이 제일 좋아? | 선호 이유를 확인한다. | |
| 어려움이 나온 경우 | 어떤 부분에서 가장 막히거나 힘들었어? | 어려움의 지점을 확인한다. | |
| 감정이 나온 경우 | 그 기분이 들 때 어떤 일이 있었어? | 감정과 상황을 연결한다. (위기 신호 시 중단) | |
| 긍정 경험이 나온 경우 | 다음에도 그런 일이 있으면 좋겠어? | 반복 의사를 확인한다. | |
| 불편한 경험이 나온 경우 | 다음에는 어떻게 되면 조금 더 좋을 것 같아? | 원하는 변화를 확인한다. | |
| 이전 이야기와 연결되는 경우 | 전에 이야기했던 때와 지금은 어떤 점이 비슷하거나 달라? | 변화를 확인한다. | |

사용하지 않을 꼬리 질문: 왜? / 그래서? / 정확히 말해봐. / 그것밖에 없어? / 누가 먼저 그랬어? / 진짜야? / 부모님에게 말했어? / 네가 잘못한 건 없어?

# 15. 상담사에게 요청할 최종 판정 항목

각 문항을 다음 기준으로 검토해 주십시오.

1. 만 9~10세(초4)가 이해하기 쉬운 문장인가 (초2~3 확장 시 별도 표기)
2. 특정 답변을 유도하는 질문은 아닌가
3. 아이가 조사·심문받는 느낌을 받을 가능성이 있는가
4. 정서적 부담이나 불안을 높일 가능성이 있는가
5. 관심사 → 일상 → 감정 순서가 자연스러운가
6. 답변을 거절할 수 있는 여지가 충분한가
7. 친구·부모·교사를 평가하거나 편을 가르게 하지 않는가
8. 감정·행동을 진단하는 질문으로 오해될 가능성이 있는가
9. 부모에게 요약 전달하기에 부적절한 사적 내용이 포함될 가능성이 있는가
10. 부적합·수정 필요 시 더 안전한 표현은 무엇인가
11. **(신설) 자·타해·학대 신호를 받은 뒤에도 이어가면 위험한 질문인가 (위기 상황 안전성)**
12. **(신설) 부모 관심사 반영 방식이 아동 신뢰를 훼손할 소지가 있는가 (섹션 13 검증)**
13. **(신설) 섹션 0 전환 규칙의 반응 문구가 만 9~10세에게 적절하고 안정화 효과가 있는가**
14. **(신설) 취침 전 문항이 수면·정서에 부정적 영향을 줄 가능성이 있는가**

---

**검토 진행 순서 안내:** 본 문서는 문항·주제·의도와 위기 전환 규칙까지 검토 대상이다. 상담사의 문항 판정과 섹션 0 검증이 끝난 뒤에 질문 주기·부모 리포트 연결·감정 힌트 3단계 표시 연동을 별도로 설계한다. 위기 전환 규칙(섹션 0)과 관련 문구는 임상 검증 통과 전까지 서비스에 적용하지 않는다.
