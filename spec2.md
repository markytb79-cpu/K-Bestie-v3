K-Bestie 관리자·서비스 통합 개발 명세 (Ralph 모드)
전체 요구사항을 한 번에 개발한다. 각 작업은 딥 인터뷰로 확정된 내용이다. 모호성 낮음. 아래 순서대로 구현하고, 로컬 빌드 검증 후 프로덕션 배포, 완료 보고.

공통 원칙: DB 스키마 변경은 최소화(불가피하면 마이그레이션 SQL 초안만 작성, 실행 전 표시). 기존 디자인 토큰(--hb-*) 유지. 요금제별 음성 방식 분기(voiceMode.ts)는 유지. 데이터는 추가 전용 원칙 유지.

1. 미션 음성 버그 수정 (최우선, 이미 일부 배포됨 — 회귀 확인)
Care Insight/Start 사용자의 미션 음성이 두 번째 턴부터 끊기는 버그. 근본 수정.

백엔드: app/api/mission/respond/route.ts에서 history를 Gemini contents로 매핑할 때 반드시 user 역할로 시작하도록 순서 보정(맨 앞이 model이면 제거 또는 앞에 user 턴 삽입). 400 에러 없이 정상 응답 확인.

프론트: hooks/useVoiceChat.ts의 오디오 재생을 매 턴 new Audio() 방식에서, useGeminiLive.ts처럼 세션 시작(사용자 인터랙션) 시 AudioContext 1회 초기화 후 재사용하는 구조로 리팩토링. 자동재생 차단 해결. 재생 실패를 조용히 삼키지 말 것.

검증: Insight 계정 미션 대화 전 구간 음성 재생, Premium 라이브 회귀 없음.

2. ai.ts 모델 상수 집중화 (Vertex 전환 선행 작업)
여러 파일(mission/respond, report/generate 등)에 하드코딩된 모델 ID·프로바이더 설정을 ai.ts 단일 지점으로 집중. 한 곳에서 프로바이더/모델을 바꾸면 전체 반영되는 구조로. 기능 변화 없음.

3. Vertex AI 전환 + AI Studio/Vertex 스위치
Vertex $300 크레딧 활용 목적. AI Studio와 Vertex를 둘 다 연결하고, 관리자 화면에서 기능 그룹별로 프로바이더+모델을 전환하는 스위치 구현. 대표가 직접 들어보고 비교·결정할 수 있게.

세 그룹 각각 독립 스위치: 그룹A 리포트·요약, 그룹B 미션 대화, 그룹C 라이브 음성. 라이브는 AI Studio(gemini-3.1-flash-live-preview)와 Vertex(gemini-live-2.5-flash-native-audio) 양쪽 다 연결해두고 스위치로 전환. 텍스트 LLM(미션·리포트)은 Vertex 대응 모델로 연결. 인증은 GCP 서비스 계정 사용. 스위치 변경은 즉시 적용하되, 진행 중이던 대화 세션은 끝까지 보호하고 다음 세션부터 새 설정 적용.

4. 리포트 차등 시스템
리포트 종류: 일간 요약, 일간 상세, 주간 요약, 주간 상세.

요금제별 제공:

Care Start: 일간 요약 + 주간 요약만. 상세 보기 탭은 자물쇠 표시 + "상세 리포트는 Care Insight로 업그레이드하세요" 안내 페이지.
Care Insight / Premium: 네 종류 전부.
주간 리포트 생성: 그 주 대화 로그 원문 전체를 AI가 재분석해서 생성(단순 이어붙이기 금지). 미션 대화는 목·금에 케이가 주말 관련 질문(주말에 뭐 하고 싶은지, 뭐 먹고 싶은지, 부모와 어떤 외식 하고 싶은지)을 자연스럽게 꺼내도록 미션 프롬프트에 반영. 주간 리포트(요약·상세 공통)에 그 답변 기반의 주말 활동 추천(외식·나들이·놀이 등) 필수 포함.

신규 개발: 일간 요약(1분 분량, 문단 하나 크기), 주간 상세, 주간 요약 조회 UI(현재 백엔드만 존재).

5. 데이터 보존 / 다운그레이드 파기
보존 기간: Care Start 6개월 고정 / Care Insight 기본 3년, 확장팩으로 1년당 월 +5,000원씩 최대 10년까지 / Care Premium 영구 기본, 사용자가 설정에서 조정 가능(하한 6개월).

유효 보존 기간 = 요금제 기본값 + 활성 확장팩. 이 값을 초과하는 데이터는 파기 대상. 모든 축소 경로(다운그레이드, 구독 해제, 확장팩 해지, Premium 기간 축소)를 이 단일 규칙으로 처리.

파기 절차: 축소 시점에 대상 데이터를 소프트 삭제(사용자에게 즉시 안 보이게, DB에는 보존) → 축소 시점 기준 정확히 30일 유예 → 자동 완전 파기. 유예 30일 안에 재구독/복구 시 데이터 연결(소프트 삭제 해제). 30일 경과 후 재구독 시 데이터 초기화.

고지: 다운그레이드 및 구독 해제 시 "데이터는 1개월 후 완전 파기됩니다" 팝업 명시(파기 범위는 경로별로 동적 계산해 표시 — 예: "6개월 초과 데이터", "3년 초과 데이터").

구현: 소프트 삭제용 deleted_at 컬럼 + 활성 데이터 필터링 뷰, 자동 파기 스케줄 잡(Supabase Cron), 유예 내 복구 로직. 파기 로직은 잘못 걸리면 데이터가 지워지므로 파기 조건·필터를 특히 신중하게 구현하고, 어떤 조건으로 무엇이 파기 대상이 되는지 명확히 주석 처리.

6. 대시보드 우패널
관리자 대시보드에서 항목(티어별 유저, 사용자별, 서비스별 탑 유저 등) 클릭 시 상세가 아래로 쌓여 스크롤이 길어지는 문제를, 오른쪽에서 슬라이드로 나오는 우패널(Helicone식)로 전환. 목록은 그대로 두고 상세만 오른쪽에 겹쳐 표시, 닫으면 사라짐.

진행: 모호성이 높은 항목이 있으면 딥 인터뷰, 아니면 Ralph 모드로 위 전체를 자율 개발·빌드·배포까지 진행하고 결과 보고. 각 작업 완료 시 무엇이 배포됐는지 항목별로 정리.
