K-Bestie 관리자 대시보드 개발 명세서
문서 버전: 초안 v1 작성일: 2026-07-12 목적: 관리자(/admin)에서 대화 내역·사용량/비용·안전 이벤트를 한 곳에서 모니터링. 특히 tier별(STT+TTS vs 라이브) 실제 비용을 아이별로 추적하고, 구글 실제 청구액과 비교 검증.

0. 전체 원칙 (제약조건)
이번 작업은 추가(additive)만 한다. 기존 미션 로직(정답 판정·게이지·황금열쇠·라운드), 리포트(gemma-4), tier 구조, safety_events, app/demo, Vercel 경로 대소문자는 건드리지 않는다. DB는 새 테이블·컬럼 추가만 하고, 위험한 변경(DROP/DELETE/컬럼 삭제)은 사전 승인. 관리자 페이지는 반드시 인증 뒤에 둔다.

1단계 — 데이터 쌓기 (최우선)
데이터는 지금부터 쌓아야 나중에 볼 수 있으므로 최우선.

1-1. 대화 내역 저장 (conversation_logs)
지금 대화가 끝나면 사라지는 문제를 여기서 같이 해결한다.

저장 대상은 미션·자유대화 둘 다, 텍스트 자막만 저장(음성 파일은 저장하지 않음 — 라이브도 휘발성 유지). 제안 스키마:

컬럼	타입	설명
id	UUID PK	
child_id	UUID FK	어느 아이
session_id	UUID	대화 세션
mode	TEXT	'mission' | 'free'
voice_mode	TEXT	'stt_tts' | 'live'
speaker	TEXT	'child' | 'k'
text	TEXT	발화 원문(자막)
created_at	TIMESTAMPTZ	
보관 정책: 개발 기간(7월 말 베타 전)까지는 무제한 보관. 베타 오픈 후에는 리포트 생성 후 7일 뒤 자동 파기. → Supabase Cron(pg_cron)으로 매일 도는 삭제 잡을 예약. (지금은 파기 잡을 비활성/보류로 두고, 베타 직전에 켠다.)

접근 권한: 대화 원문은 기본적으로 누구도 못 봄. 개발 기간에 한해 관리자(형진)만 /admin에서 확인. RLS는 service-role만 read, 부모/아이 접근 차단.

1-2. 사용량·비용 기록 (usage_events)
아이별로 세밀하게 기록(누가 비용을 많이 쓰는지 파악용).

컬럼	타입	설명
id	UUID PK	
child_id	UUID FK	
tier	INT	1/2/3
voice_mode	TEXT	'stt_tts' | 'live'
kind	TEXT	'stt' | 'tts' | 'live_audio' | 'llm'
duration_sec	NUMERIC	라이브·STT 대화 시간(초)
char_count	INT	TTS 글자 수
token_in / token_out	INT	토큰 수(있으면)
est_cost_krw	NUMERIC	우리 단가표로 계산한 예상 원가
created_at	TIMESTAMPTZ	
핵심: 라이브는 세션 열린 시각~닫힌 시각으로 duration_sec 기록(라이브 비용의 근거). STT+TTS는 각 호출의 시간/글자수 기록.

1-3. 단가표 (코드 상수, 검토 필요)
검색으로 확인한 최신 기준 초안 — 실제 청구액과 비교하며 조정 예정:

라이브 오디오: 오디오 초당 25토큰(입/출력 각각) 기준. 실측으로는 대략 분당 60~80원(우리 테스트: 30개 목소리 5~7.5분에 약 400원 = 분당 55~80원과 일치).
Wavenet TTS: 100만 자당 약 $4.
STT: 별도 단가(확인 필요). 이 값들은 명세서 확정 시 최신 공식 단가로 다시 채운다.
2단계 — 관리자 페이지 + 인증
2-1. 인증
경로는 /admin (기존 앱 안 숨겨진 경로). 관리자는 현재 형진 1명(확장은 약 1년 뒤 예정이므로 지금은 단순하게). 인증 방식: 구글 계정 로그인 + 관리자 이메일 화이트리스트. 화이트리스트에 등록된 이메일로 구글 로그인한 경우에만 /admin 접근 허용. (실제 이메일은 문서에 안 적고 구현 시 환경변수/코드에 넣음.)

→ 열린 질문: 지금 앱에 구글 OAuth(로그인)가 이미 붙어있는지 클로드가 먼저 확인해야 함. 이미 있으면 재사용, 없으면 관리자용으로만 Google OAuth를 신규 연결.

2-2. 화면 구성 (탭)
/admin 안에 탭 3개:

대화 내역: 아이 선택 → 세션별 대화 원문 타임라인(누가/언제/뭐라고). 개발 기간 한정 노출.
사용량·비용: 아이별·tier별·기간별 집계. 우리 단가표로 계산한 예상 원가 표와 차트. (3단계 붙기 전까지는 우리 데이터 기준.)
안전 이벤트: safety_events 조회 — 누가/언제/몇 시/어떤 카테고리/뭐라고 말했는지. 폭력·자해·위협·부적절접촉·방임.
3단계 — 구글 빌링 연동 (형진 설정 필요)
목표: 우리 예상 원가 vs 구글 실제 청구액을 같은 기간 기준으로 나란히 비교.

방식: GCP Billing을 BigQuery로 Export(공식 기능) → 우리 서버가 BigQuery를 조회해 날짜별 실제 비용을 가져와 대시보드에 표시. (빌링 데이터는 실시간 아님, 하루 정도 지연 — 사후 비교엔 문제없음.)

형진이 GCP 콘솔에서 할 일 (한 번만, 방법 안내)
어느 프로젝트에 Gemini API가 붙어있는지 확인: 화면에서 본 "K-Bestie Gemini Project"가 그 프로젝트로 보임. AI Studio에서 발급한 API 키가 이 GCP 프로젝트에 연결돼 있어야 빌링이 잡힘.
GCP 콘솔 → 결제(Billing) → 결제 내보내기(Billing export) → BigQuery export 탭.
"상세 사용 비용 데이터(Detailed usage cost data)" 를 켬(이게 서비스별/AI 비용 추적에 필수).
Export를 담을 BigQuery 데이터셋을 만들거나 선택.
저장하면 이후 매일 자동으로 비용 데이터가 BigQuery에 쌓임.
우리 서버가 그 BigQuery를 읽도록 서비스 계정 키를 발급해 앱 환경변수에 등록.
→ 각 단계는 실제 콘솔 화면 보면서 내가 하나씩 짚어줄게(스크린샷 주고받으며). 이건 클로드가 대신 못 눌러(네 결제 권한 필요).

→ 열린 질문: 형진에게 GCP 결제 관리자 권한이 있는지. 없으면 3단계 보류, 1~2단계 먼저 완성.

4. 클로드 딥 인터뷰용 열린 질문 목록
현재 앱에 구글 OAuth 로그인이 이미 구현돼 있나? (관리자 인증 재사용 가능 여부)
대화 세션(session_id)이 지금 코드/DB에 이미 정의돼 있나? 미션·자유대화가 각각 세션을 어떻게 구분하나?
라이브 세션의 시작/종료 시각을 정확히 잡을 훅(useGeminiLive의 connect/close)이 있나? duration 계산 가능한가?
STT+TTS 경로에서 각 호출의 글자수·시간을 지금 로그로 얻을 수 있나?
safety_events에 지금 어떤 컬럼이 있고, 대시보드에 필요한 정보(시각·아이·원문)가 다 들어있나?
usage_events 기록을 어디서 남길까 — API 라우트(서버)에서? 그래야 클라이언트 조작을 막음.
Supabase Cron으로 자동 파기 잡을 지금 만들되 비활성으로 둘까, 베타 직전에 만들까?
5. 진행 순서 (합의된 방식)
이 명세서 확인 → 클로드와 딥 인터뷰(위 열린 질문 해소) → 모호성 제거.
1단계(테이블 2개 + 기록 로직) 착수. DB는 SQL 초안 먼저 검토 후 실행.
2단계(인증 + /admin 3탭).
3단계(형진 GCP 설정 → BigQuery 연동 → 비교 화면).
